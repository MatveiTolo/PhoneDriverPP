# PhoneDriverPP

Управление Android телефоном через ADB из командной строки с поддержкой анализа экрана и AI-автоматизации.

Проект рассчитан на локальный запуск Ollama на вашем ПК. Перед стартом обязательно подготовьте все зависимости ниже, иначе AI-часть проекта не сможет работать.

## Быстрый старт

### 1. Установите ADB

Скачайте [Platform Tools] и добавьте `adb` в PATH.

### 2. Установите и запустите Ollama на своем ПК

Ollama должна работать на том же компьютере, где запускается проект.

Установите Ollama с официального сайта, затем запустите сервис:

```bash
ollama serve
```

После этого заранее загрузите модель, которая используется в проекте:

```bash
ollama pull qwen3-vl:4b
```

Именно модель `qwen3-vl:4b` нужна для анализа экрана и принятия решений по действиям.

### 3. Подготовьте телефон

- Включите **"Режим разработчика"** (нажмите 7 раз на "Номер сборки")
- Включите **"Отладку по USB"**
- Подключите телефон кабелем и нажмите **"Разрешить"**

### 4. Установите Python-зависимости

```bash
pip install -r requirements.txt
```

Дополнительно для OCR требуется установленный Tesseract OCR в системе.

### 5. Запустите проект

```bash
git clone https://github.com/ВАШ_АККАУНТ/PhoneDriverPP.git
cd PhoneDriverPP
python main.py
```

Если нужен только базовый ADB-режим без AI-логики, можно запускать:

```bash
python driver.py
```

## Что должно быть запущено до старта

- Ollama на локальном ПК: `ollama serve`
- Модель `qwen3-vl:4b`: `ollama pull qwen3-vl:4b`
- ADB и подключенный Android-телефон
- Tesseract OCR для распознавания текста на экране

## Настройка по умолчанию

В [config.py](config.py) по умолчанию указан локальный адрес Ollama:

```python
OLLAMA_HOST = "http://127.0.0.1:11434"
OLLAMA_MODEL = "qwen3-vl:4b"
```

Если Ollama у вас работает на другом адресе, измените `OLLAMA_HOST` в [config.py](config.py).
